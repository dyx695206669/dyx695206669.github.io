String

1.String不可变(GPT)

在Java中，字符串（java.lang.String）是不可变的，也就是说一旦字符串对象被创建后，它的值就不能被修改。这意味着对字符串进行修改操作时，实际上是创建了一个新的字符串对象。

2.String不可变的好处(GPT)

为什么要设计字符串为不可变的呢？这是因为字符串的不可变性带来了一些好处：

  • 线程安全：由于字符串的不可变性，多个线程可以安全地共享字符串对象，而无需担心对象的状态被修改。
  • 缓存利用：字符串池的存在可以重复利用字符串常量，避免创建大量相同值的字符串对象，减少内存消耗。
  • 安全性：不可变字符串可以作为参数传递给不可信的代码，不用担心它们的值被篡改。

2.JVM

​ ①编译→类加载到内存→运行时数据区→方法区

静态static

​ ①static修饰的成员变量属于类，该成员变量只在内存中存储一份(值可修改，但地址不变)，可以被共享访问和修改。
​ ②访问格式：类名.静态成员变量
​ ③无static修饰的被称为实例成员变量，存在于每个对象中，访问格式：对象名.实例成员变量
​ ④静态方法不能使用this，实例方法相反。
​ ⑤对于工具类，主程序直接调用类的静态方法，即不需要创建对象，因此可以私有化构造器。
​ ⑥代码块，永远比主方法先执行
​ ⑦关于任意方法中不能声明静态变量——

​ 从意义上：静态变量属于类，可以通过类直接调用，定义在方法中没意义。
​ 从生命周期上：静态变量是在类加载时初始化，并在整个程序运行期间存在。方法是属于类的行为或操作，只有在调用方法时才会在堆栈中创建方法的执行环境，并在方法执行完毕后销毁。由于静态变量的生命周期长于方法的执行周期，如果允许在方法内部声明静态变量，那么每次方法执行时都会创建新的静态变量，这与静态变量的设计初衷相违背。

单例设计模式

​ ①目的：确保一个类只有一个对象
​ ②步骤：私有化构造器，定义公开的静态变量用于存储对象，提供返回单例的方法(判断是否初始化)。

Final

​ ①修饰类——不可继承；
​ ②修饰方法——仅该方法不可重写；
​ ③修饰变量——基本数据类型，常量，只能被赋值一次，声明时可先不赋值；
​ 引用数据类型，地址不变，属性可变，声明时可先不赋，建议在构造方法中赋值；
​ ④使用static final修饰的变量，声明时即需要赋值，不可修改，通常全大写

继承

​ ①子类不能继承父类的 构造器(自己有)，但是可以调用(super())。
​ ②子类不能继承父类的 私有成员变量/方法，即不能直接访问/调用。
​ 但子类可以通过getter、setter方法访问父类的私有成员变量，也可以暴力访问(反射)。
​ 同时子类可以通过调用公共方法间接触发父类的私有方法执行(如该方法中含有父类的私有方法)。
​ ③子类不能继承父类的 静态成员变量/方法(公开，并不算继承)，但可以通过类直接调用。
​ ⑤子类不能重写父类的 构造方法(类名字压根不同)、私有方法、静态方法、final修饰的方法。
​ ⑥子类不能重写父类的final方法，但继承父类的final成员变量和方法。

多态

​ ①成员变量不能多态
​ ②多态的弊端：无法直接调用子类自己的方法
​ 需要强转类型，把父类转为子类，同时，为了避免转错对象，需要先进行判断
​ if(父类对象 instanceof 子类){
​ 子类 子类对象 = (子类) 父类对象
​ }

抽象类
① 对于一个方法，被所有继承的子类重写，则这个方法本身不具有实现意义，需被定义为抽象类
②因为不需要被实现，所以抽象方法不需要方法体
③抽象方法一定在抽象类中，但抽象类不一定需要抽象方法
④抽象类具有构造器，但不能创建对象
⑤继承抽象类的子类，必须重写所有的抽象方法，否则自身也必须是抽象类
⑥final类不可被继承，final方法不能被重写，因此abstract和final不能同时出现。

模板方法设计模式

​ ①模板方法设计模式解决重复代码的问题。
​ ②步骤：设计一个抽象类，定义一个final模板方法，存放公共信息；和一个抽象方法，交给子类去实现。

接口

​ ①接口不能创建对象
​ ②接口的成员变量为常量(定义即赋值)，接口的方法默认为抽象方法(无需方法体)
​ ③实现类必须重写接口的所有抽象方法，否则定义为抽象类

内部类

​ ①匿名内部类，一种特殊的局部内部类，匿名指的是不需要为其命名
​ ②形式：new 类/接口(形参){ 类体(一般为方法重写) };
​ ③匿名内部类本质是一个子类，并立即创建一个新的子类对象
​ ④匿名内部类通常作为一个参数传输给方法

枚举

​ ①枚举第一行必须是全员大写的名称，常量，记录都是每个枚举类的对象

泛型

1.认识泛型

​ ①定义 类、接口、方法时，同时声明了一个或多个类型变量，称为泛型

​ ②避免强制转换类型或其它异常

2.泛型修饰类与方法

​ ①自定义泛型类，可以存储指定的数据类型：

​ 修饰符 class 类名<类型变量…>{}

​ ②泛型方法：接收什么类型的数据，就返回什么类型的数据：

​ public static T getByT(T t){ return t;}

3.通配符
  • 通配符 extends 父类 –上限，即只能传入子类和父类
  • 通配符 super 父类 –下限，即只能传入父类和父类的父类

常用API

1.重写原本属于Object类的toString和equals()
2.clone() 对象克隆
  • 浅克隆，即Object类的克隆，复制值/地址
    • 记得先实现Cloneable接口，同时返回类型为Object，需要强转
    • 一旦引用类型的值发生改变，原对象会发生改变，同时带动克隆对象发生改变
  • 深克隆，重写Object类的克隆，引用数据类型会创建新对象，导入旧数据。
    • 先调用原来的浅克隆，将生成的对象强转成自己需要的类型，新建引用数据类型对象，利用get、set方法存入数据，最后返回需要的数据类型
3.Objects类的equals()
  • NullPointerException，空指针异常，当调用方法的对象为null时出现
4.包装类
  • 包装类，把基本数据类型包装成对象
5.StringBuilder

6.StringJoiner
  • StringJoiner也可以高效率拼接字符，还可以设置前缀、分隔、后缀符号

JDK8.0新特性

1.Lambda表达式
  • Lambda表达式是JDK8.0的新特性，用于简化匿名内部类的代码写法
  • 注意：Lambda表达式只能简化函数式接口(有且仅有一个抽象方法的接口)的匿名内部类
2.Lambda省略规则
  • 参数类型可以不写
  • 如果只有一个参数，参数类型可省，()也可省
  • 如果Lambda表达式的方法体代码只有一行，大括号可省，分号必省，return必省
3.方法的引用
  • 静态方法的引用
    • 如果一个Lambda表达式里只调用一个静态方法，并前后形参一致，可通过“ 类名::静态方法(不带括号) ”进行进一步省略。
  • 实例方法的引用
    • 如果一个Lambda表达式里只调用一个实例方法，并前后形参一致，可通过“ 对象名::实例方法(不带括号) ”进行进一步省略。
  • 特定类型方法的引用
    • 如果一个Lambda表达式里只调用一个实例方法，并且前面参数列表中的第一个参数是作为方法的主调，后面的所有参数都是作为该实例方法的入参，可通过“ 类型::实例方法(不带括号) ”进行进一步省略。
  • 构造器引用
    • 如果一个Lambda表达式里只是创建对象，并且前后参数一致，可以通过“ 类名::new ”进行进一步省略。

常见算法

1.冒泡排序

​ ①原理：每次从数组中找到最大值放在数组后面

​ ②外循环：需要的轮数i = 数组长度-1

​ ③内循环：第i次的比较次数j = 数组长度-i-1

2.选择排序

​ ①原理：每轮选择一个位置，开始找出后面的较小值与该位置交换

​ ②外循环：需要的轮数i = 数组长度-1

​ ③内循环：第i次的比较次数j+1：数组长度

3.二分查询

​ ①前提：数组必须是有序的

​ ②核心思想：每次排除一半的数据

正则表达式Regex

1.匹配单个字符

​ ①[abc] 只能匹配a、b、c其中一个字符

​ ②[^abc]除了a、b、c以外的一个字符

​ ③[a-zA-z] 只能是a-z和A-Z中的一个字符

​ ④[a-z&&[^bc]] 包括a-z，除了b和c中的一个字符

​ ⑤[a-zA-Z0-9]字符数字，一个字符

​ ⑥”.”代表任意字符，只能匹配一个字符

2.特殊符号(匹配单个字符)

​ ①\d 代表数字

​ ②\s 代表空白字符

​ ③\S 代表非空白字符

​ ④\w 代表字符数字下划线

​ ⑤\W 代表非字符数字下划线

3.特殊符号(代表次数)

​ ①?代表出现一次或者零次

​ ②*代表出现多次或者零次

​ ③+代表出现多次或者一次

​ ④{n}代表出现正好n次，{n,}代表出现至少n次，{n,m}代表出现n次到m次之间

​ ⑤(?i)代表出现忽略大小写

4.常用方法

​ ①replaceAll(String regex, String newStr)——按照正则表达式匹配的内容进行替换

​ ②split(String regex)——按照正则表达式匹配的内容进行分割字符串，返回一个字符串数组
